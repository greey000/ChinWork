# 教學醫院課程雷達 — 設定教學

這是一個「一次設定、之後全自動」的課程彙整網站：
每天由 GitHub 免費幫你排程執行一次抓取程式，把新課程寫進 `data/courses.json`，
再由 `index.html` 讀取顯示成一個網頁。全程不需要自己寫程式，也不需要自己的伺服器。

---

## 你需要準備的東西

- 一個 GitHub 帳號（免費）：https://github.com/signup

---

## 步驟一：建立你自己的 Repository（專案倉庫）

1. 登入 GitHub，右上角「+」→「New repository」
2. Repository name 隨意，例如 `course-radar`
3. 選 **Public**（GitHub Pages 免費方案需要 Public repo）
4. 按「Create repository」

## 步驟二：把這個資料夾的檔案上傳上去

在你剛建立的 repo 頁面：
1. 點「Add file」→「Upload files」
2. 把這次收到的所有檔案（含 `.github` 資料夾）拖曳上傳
   - 如果網頁介面不讓你上傳隱藏的 `.github` 資料夾，改用「uploading an existing file」逐一建立，
     或改用 GitHub Desktop（免安裝指令、圖形介面：https://desktop.github.com/）把整個資料夾同步上去，會更順利
3. Commit（提交）

## 步驟三：打開 GitHub Actions 的執行權限

1. 進入 repo 的 **Settings** → **Actions** → **General**
2. 找到「Workflow permissions」，選擇 **Read and write permissions**
3. 儲存

（這一步是讓排程程式抓完資料後，能夠自動把結果寫回這個 repo）

## 步驟四：手動先跑一次，確認有抓到資料

1. 進入 repo 的 **Actions** 分頁
2. 左側選「Scrape Courses」
3. 右邊按「Run workflow」→「Run workflow」
4. 等 1-2 分鐘，點進去看執行紀錄（log）
   - 如果看到「找到 0 個候選項目」，代表該來源的網頁結構需要調整，見下方「常見問題」
5. 成功的話，`data/courses.json` 會被自動更新並多一個 commit

## 步驟五：開啟 GitHub Pages，讓網頁可以被瀏覽

1. 進入 repo 的 **Settings** → **Pages**
2. Source 選擇「Deploy from a branch」，Branch 選 `main` / `/(root)`
3. 儲存，等 1-2 分鐘
4. 網址會顯示在同一頁面，通常是 `https://你的帳號.github.io/course-radar/`

之後每天 GitHub 會自動在台灣時間早上 9 點跑一次，你完全不用管它。

## 之後想新增醫院/學會來源，只要編輯 `sources.json`

```json
{
  "name": "顯示用的來源名稱",
  "url": "該機構的課程/公告列表頁網址",
  "list_selector": "body"
}
```

加一筆存檔、commit，下次排程執行就會一併抓取。`list_selector` 先保持 `"body"` 即可，
如果抓到太多雜訊（選單、頁尾）再依下方方法調整。

## 常見問題

**Q: log 顯示「找到 0 個候選項目」或抓到都是選單雜訊怎麼辦？**
用瀏覽器打開該網頁，按右鍵「檢查」（Inspect），找到實際列出課程清單的區塊
（通常是一個 `<table>`、`<ul>` 或 `<div class="...">`），把它的 class 或 id
填進 `sources.json` 的 `list_selector`，例如 `"list_selector": "div.content"`。
如果不確定怎麼看，把該頁面的 HTML 原始碼片段貼給我，我可以幫你抓出正確的 selector。

**Q: 網站需要登入才能看到課程怎麼辦？**
這類來源建議跳過，改用人工方式定期查看，或詢問該機構是否有公開的課程月曆/RSS。
自動抓取只適合「公開瀏覽即可看到」的頁面。

**Q: 想要更漂亮的分類（像復健科進修雷達那個網站有「徒手/動作」「疼痛/介入」分類）？**
目前版本是抓連結標題自動比對「新舊」，還沒有分類功能。之後可以在 `data/courses.json`
裡幫每筆資料加 `category` 欄位（可以用關鍵字比對，或用 AI 自動分類），
我可以之後幫你加這個功能。

## 心得表單：讓無法自動核發證明的課程也能申請學分

有些課程（例如 NUR 課程、線上讀書會）不會自動核發結業證明，需要學員填寫心得，
你再統一拿這些心得去跟認證單位申請、上傳學分。這部分用 **Google 表單 + Google 試算表 +
一小段 Apps Script** 完成，跟前面 GitHub 那套完全獨立，不用架伺服器。

### 設定步驟（一次性）

1. 開啟 `credit-form-setup/Code.gs`，把裡面的 `ADMIN_EMAIL` 改成你自己收通知信的信箱
2. 到 https://script.google.com/ 建立新專案，把整份 `Code.gs` 內容貼進去
3. 上方函式選單選 **setup**，按執行（Run）
   - 第一次執行會跳出 Google 授權畫面，因為是你自己的腳本會顯示「未經驗證」，
     這是正常現象，點「進階」→「前往...(不安全)」→「允許」即可
4. 執行完成後，點下面「執行紀錄」，會印出三個網址：
   - **表單填寫網址**：給學員填的，把這個貼到下一步的 `reflection-required.json`
   - **表單編輯網址**：你之後想改欄位用這個
   - **回覆試算表網址**：你審核學分用這個，已經幫你多加了「審核狀態」「已申請學分」「備註」三欄

之後完全自動：學員填完表單 → 系統自動寄確認信給學員 → 自動寄通知信給你，
提醒有新回覆進來要審核。你只要固定去試算表看、勾選審核狀態，
定期把「通過」的名單匯出去跟認證單位申請學分即可。

### 設定哪些課程要顯示「請填心得」按鈕

編輯 `reflection-required.json`：

```json
{
  "form_url": "貼上面 setup 執行後印出的表單填寫網址",
  "match_sources": ["臺灣醫學教育學會-線上書院"],
  "match_keywords": ["NUR"]
}
```

- `match_sources`：來源名稱完全符合 `sources.json` 裡的 `name`，該來源底下**所有課程**都會顯示按鈕
- `match_keywords`：課程標題只要包含這些關鍵字（不分大小寫）就會顯示按鈕，適合像「NUR」這種課程代碼
- 存檔後，`index.html` 打開時會自動讀取這份設定，在符合條件的課程旁邊顯示
  「📝 無法自動核發證明，上完課請填心得申請學分」按鈕，連到你的 Google 表單

### 提醒

- 表單會收集學員姓名、Email 等個資，建議在表單描述裡加一句「本資料僅用於學分申請，不做其他用途」
- 如果之後同一批課程要開放很多期，可以把 `match_keywords` 設得更精準（例如加上年度代碼），
  避免不小心把不相關的課程也標記上

## 讓「已認列學分」也反映到課程清單，並自動整理簽到清單

你的需求是：不是每堂課都算學分，只有「學員有回覆心得」的才算，而且這件事要同時反映在：
1. 網站上的課程清單（讓大家看得到哪些課已經有人完成、學分認列中）
2. 你申請學分要用的「學員簽到清單」

做法是這樣分工，**學員名單只留在你自己的試算表裡，網站上只公開總人數，不公開姓名**（保護個資）：

### 1. 設定「課程名稱」預先帶入，確保名稱能對得起來

學員自己在表單打的課程名稱，可能跟網站上的標題有落差（漏字、簡稱），這樣後面要統計
「這門課有幾個人回覆」就會對不起來。解法是讓學員從課程清單點按鈕過去時，
自動把正確的課程標題帶入表單：

1. 打開你的 Google 表單（`credit-form-setup/Code.gs` 執行後印出的「表單編輯網址」）
2. 「課程名稱」欄位右上角三個點 → **取得預先塞入的連結**
3. 隨便打一個課程名稱送出，網址列會出現類似
   `...viewform?usp=pp_url&entry.987654321=測試課程` 這樣的網址
4. 把中間 `entry.` 後面那串數字（例如 `987654321`）填進 `reflection-required.json` 的
   `course_name_entry_id`

### 2. 部署「已認列人數」的公開查詢網址

1. 回到 Apps Script 專案，右上角「部署」→「新增部署作業」
2. 類型選 **網頁應用程式**
3. 執行身分：**我**；誰可以存取：**所有人**
4. 部署後複製網址（類似 `https://script.google.com/macros/s/xxxx/exec`）
5. 貼到 `reflection-required.json` 的 `status_api_url`

這個網址只會回傳「課程名稱 + 已通過人數」，**不含任何學員姓名或Email**，可以放心公開。
存檔後，`index.html` 打開時會自動去讀，在對應課程下面顯示
「✅ 已有 N 人送出心得，學分認列中」。

### 3. 審核 + 產生簽到清單

平常流程：
1. 學員填表單 → 進你的回覆試算表，`審核狀態` 欄位預設空白
2. 你看過心得內容，覺得可以算學分的，把該列 `審核狀態` 填成「**通過**」
3. 要送學分申請前，回到 Apps Script 專案，函式選單選 **generateSignInSheets**，執行
4. 它會依「課程名稱」分組，自動在同一份試算表裡建立/更新
   `簽到_課程名稱` 分頁，欄位是序號、姓名、服務單位／職稱、Email、上課日期
   —— 這份就是你要送給認證單位的學員簽到清單，可以直接匯出或列印
5. 網站上「已有 N 人送出心得，學分認列中」的數字，也是統計同一批「審核狀態＝通過」的人數，
   兩邊會保持一致

之後這三步（審核 → 產生簽到清單 → 網站數字自動更新）可以固定每次要申請學分前做一輪，
不用另外維護第二套名單。

## 不定期收到課程電子檔（PDF/email/圖片），怎麼手動加進網站

如果課程資訊是別人傳來的電子檔或訊息，不是掛在一個會自動更新的網頁上，
不適合寫爬蟲，改用手動加入 `data/manual-courses.json` 這個檔案，
排程程式完全不會去動它，網站會自動把它跟自動抓到的課程合併顯示。

### 操作步驟（都在 GitHub 網頁上做，不用裝軟體）

1. 如果課程資訊是 **PDF 或圖片檔**，要先讓它有一個網址：
   - 最簡單：到你的 repo → `manual-files` 資料夾（沒有就用「Add file → Upload files」建立）
     把檔案上傳上去，之後可以用
     `https://你的帳號.github.io/repo名稱/manual-files/檔名.pdf` 這個網址存取
   - 如果課程資訊本來就是 **一個網頁連結**（例如對方寄給你一個報名網址），
     就不用上傳檔案，直接用那個網址即可
2. 到 repo 裡打開 `data/manual-courses.json`，按右上角鉛筆圖示「編輯」
3. 仿照裡面的範例，加一筆新的（記得前一筆結尾要加逗號 `,`）：

```json
{
  "source": "課程主辦單位名稱",
  "title": "課程完整名稱",
  "url": "課程網頁連結或你剛上傳的 PDF 網址",
  "first_seen": "2026-07-14"
}
```

4. 記得刪掉一開始的範例那筆（或改成真的資料）
5. 下方「Commit changes」直接送出，網站重新整理就會看到

### 如果不想自己編輯 JSON

之後有新的課程電子檔，你也可以**直接把內容（文字或截圖）貼給我**，
我幫你把格式整理成上面那樣的 JSON 區塊，你只要複製貼到 GitHub 網頁上、按 Commit 就好，
不用自己想欄位怎麼填。
